# Skill Library Forge Audit — 2026-06-25

Deep-dive across all 52 skills to plan a behavioral test-and-improve ("forge") pass on each, following the pattern proven on **prose-polish** (clean-room A/B → diagnose → harden → verify). Surveyed by 7 parallel audit agents reading every SKILL.md + references.

## The headline finding

The prose-polish discovery — a skill that **fabricates content on low-source inputs** — is **not a one-off**. It is a structural failure *family* that recurs wherever a skill must produce a confident artifact from thin/ambiguous input and lacks a hard "verify-or-flag, never invent" gate. The proven remediation is the **prose-polish v1.2 Fidelity Firewall** (every concrete specific must trace to the source; fallbacks = sharpen / `[flag]` / reduce-claim). Porting that pattern is the single highest-leverage action across the library.

**Eval-design warning (from skill-forge v1.2):** the fabrication-class skills are value-shape **(c) gating/correctness**, NOT (a) artifact-quality. They MUST be forged with a **trigger-conditional, ground-truth-verification** eval — a default mean-lift artifact eval would *reward the confident fabrication and erode the gate* (skill-forge's documented cardinal error). Flag this on every (c) bead.
