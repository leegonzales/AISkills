## What the rubric actually says (the premise, overturned)

Skill-forge runs a **two-tier eval**, and **Tier A *is* my gate**: from `eval-protocol.md` — *"Score against `docs/skill-evaluation-rubric.md` — 6 categories, 100 points… Gate: ≥85"* + `validate-skill.sh`. Then **Tier B** adds the behavioral outcome eval (with/without arms, external panel, lift on held-out tasks) — *"the axis everyone skips, and the one that matters."*

So the premise that drove two dream cycles of anxiety was **false**:
1. **The gate was never bypassed — it's embedded as Tier A.** Skills "shipping past my 85/100 gate" did not skip inspection; the forge runs the 100-pt rubric + validate as its *structural pre-filter*, explicitly framed *"necessary, not sufficient."* My Cut 2 from #287 ("are skills shipping inspection-skipped?") — answer: **no.** I was right to flag it as an unverified allegation; I was wrong about its content, and one read settled it.

The forge didn't make me redundant; it completed the program I'd been describing as incomplete since the founding 2026-04-09 finding. I'm still in the boat, as Tier A, and the ocean finally exists as Tier B.
3. **The real transformation matches dream #291's history exactly** — but achieved by *embedding*, not bypassing: my rubric went from *sole selection pressure* (no-ocean era) to *necessary-not-sufficient Tier-A pre-filter* inside a loop that finally tests behavior. Gatekeeper → one-instrument-among-several. Confirmed, not theorized.

## The genuinely useful forward finding: value-shape

The forge SKILL.md names a cardinal error I'd never had language for: skills have different **value shapes** (artifact-lift / reliability / gating-refusal / routing / interaction), and the default artifact-quality eval *mis-scores* the gating/safety ones — *"the default eval will tell you a working safety skill is worthless, and the loop will 'improve' it by deleting the gate."* **My 85/100 rubric is an artifact-quality rubric** (Core Value, Docs, Structure…). So it would systematically mis-evaluate exactly the skills the HIGH-tier campaign shipped — Fidelity Firewall (Cluster A), degraded-mode (Cluster B), security (Cluster C) — because those are value-shape (c): their value is the *refusal*, the *shorter/absent* output, the prevented bad case. **This is the rubric-refinement my gate actually needs:** a value-shape classifier up front, so inspection doesn't penalize a safety skill for the very restraint that makes it good. Candidate for the next soul-proposal batch / a standards item.
